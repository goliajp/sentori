# v1.1 polish backlog — P2 items left for incremental polish

Everything in this list is real but **doesn't block ship**. P0
(4 items, security/reliability) and P1 (~25 items, hardness +
visible UX gaps) are done. The items below are the "perfectionist"
tier — fix opportunistically when you're already in the file.

## Server

- `culprits.rs` GitHub HTTP timeouts (10 / 15 / 20s) are magic
  constants per fn; no env override. Enterprise proxies with > 10s
  latency would currently fail. Consider `SENTORI_GITHUB_TIMEOUT_MS`
  fallback to current defaults.
- `culprits.rs` manual `attach` returns 201 even when the GitHub
  metadata fetch fails — author/message null in DB. Behavior is
  intentional (manual attach should not require GitHub be reachable)
  but the response could carry a `"metadataFetchFailed": true` hint
  so the dashboard shows "couldn't enrich" instead of empty fields.
- `live_debug.rs` arm/disarm/stream protected by
  `require_project_in_org` middleware. Adding a defensive
  handler-level check (`is_admin_for_project(pool, &caller, project_id)`)
  would be defense-in-depth — middleware misconfiguration during a
  future refactor would otherwise expose these endpoints with no
  fallback.
- Error message friendliness: several handlers return generic
  `AppError::Internal(internal-string)` whose body is just
  `{"error": "internal"}`. Consider promoting more specific
  `Unconfigured(...)` codes for known operator-fixable failures.
- `vitals.rs:140` `unwrap_or((0,0,0))` for missing per-route TTFD
  data quietly returns zeros instead of nulls — a route with TTFD=0
  means the same as "no samples." Dashboard already shows "—" for
  the no-samples case via `ttfdSamples > 0` check; this is
  cosmetic.

## SDK (@goliapkg/sentori-react-native)

- `launch-crash-guard.ts` 5+ `catch {}` blocks intentionally swallow
  AsyncStorage glitches. By design — the guard must never block
  init. But a console.warn under `__DEV__` only would help debugging.
  Need to weigh against the SDK's "no console output" convention.
- `pre-crash-sentinel.ts` `PreCrashChannel` union claims to support
  future native channels but only `'js-frame-budget'` is wired.
  Either remove the union now (one-string-type) or implement at
  least a native counterpart.
- `replay.ts` SVG node rendering: text fontSize calculation
  (`Math.min(14, Math.max(8, h * 0.6))`) is a useful heuristic but
  the constants 14/8/0.6 deserve a comment.
- Public API option shape consistency: `replay: 'off' | 'wireframe'
  | { hz, mode }` mixes string + object, asymmetric vs.
  `sampleProfiler: boolean | { sampleMs, flushMs }`. Pick one shape
  for all v1.1 capture options.
- `sample-profiler.ts` doesn't sample frames during busy JS. Document
  this clearly in JSDoc on `startSampleProfiler` — the v1.2 Hermes
  off-thread sampler is the fix. (Partially done in the source
  comment.)

## Dashboard (web/)

- AttachmentGallery hover transitions on tiles and lightbox nav
  buttons feel snappy — no `transition-colors` / easing.
- LiveDebugView status indicator (●) lacks animated pulse for the
  "connected" state. Linear-style live dot would be nicer here.
- LiveDebugView table header styling: `text-fg-muted uppercase
  tracking-wider` is missing vs other section headers; inconsistent
  density.
- CulpritSection collapsed state has no visual badge for "auto-
  detect pending" vs "unattributed" — both render as plain empty.
- CulpritSection rows: alignment varies between (author span,
  auto-label chip, open link, revert button). Tightening to a
  uniform grid would clean it up.
- IssueDetailView Tabs active state: currently `text-accent` only —
  Linear uses an underline indicator that survives focus changes.
  Adding `relative` + `::after` pseudo-element would do it.
- EventPicker chevron buttons lack a hover background (only text
  color changes — weak affordance).
- VitalsView "No samples" empty state for cold_start uses a plain
  div; AttachmentGallery's `InfoBox`-style container would unify the
  pattern.
