## L3a — Hot plan: 当前 checkpoint = **v0.9.0** 第一项

每一步必须有检测命令。Test-driven。无分叉。

### Step 1 (#11): GraphQL operation auto-naming

1. **read** `sdk/react-native/src/handlers/network.ts` — 摸现网络 instrumentation 形状
   - check: `wc -l src/handlers/network.ts` 输出非空
2. **edit** `src/handlers/network.ts` — POST 请求体小 (< 8 KB) 且 content-type 含 `graphql` 或 URL 含 `/graphql` 时尝试解析 body 中的 `operationName` 字段；找到则 breadcrumb name 改为 `graphql/<operationName>`，并把 `gql.operation` 入 breadcrumb data
   - check: `bun run typecheck` 退出码 0
3. **add** `__tests__/network-graphql.test.ts` — 3 个 case：(a) 含 operationName 的 POST → breadcrumb 重命名；(b) GET /graphql → 不重命名；(c) 解析失败 → 退化到原行为
   - check: `bun test __tests__/network-graphql.test.ts` 全绿
4. **update** `src/init.ts` 加 `capture.network.graphql` 配置项（默认 true）
   - check: `bun run typecheck`
5. **run full test suite**: `bun test`
   - check: existing 60 tests + 新 3 测试都过
6. **commit**: `sdk/react-native: #11 GraphQL operation auto-naming`

完成 step 1 后 mark task as completed，开始 step 2 (#12)。

## L3b — Cold plan (v0.9.0 剩余)

每项粗述，不写 step。Hot plan 升级时再细化。

- **#12 Rage tap** — 一个 `<RageTapCapture>` 组件包 children，gesture 监听 800ms 内 ≥ 3 次点击同元素 → emit `ui.multiClick` breadcrumb。
- **#13 Feature Flag** — `sentori.setFeatureFlag(name, value)` / `clearFeatureFlag(name)`，map 形态，每事件 ride along；server 加 `flags` jsonb 列 + 索引；dashboard 加 filter chip。
- **#14 measureFn** — `sentori.measureFn(name, fn)` 包 span 系统的薄壳。
- **#5 Velocity** — server cron 每 5 分钟扫 issues，比较 30m count vs prev 30m，跨阈值（3x / 5x）走 Slack/Linear；新事件 + bundle id 上下文。需要 `notification_recipients` 表加 `on_velocity` 列。
- **#6 Moments** — `sentori.startMoment(name, opts).end() / .fail() / .abandon()` + checkpoint；core 加 `MomentHandle`；server span `op = sentori.moment`；dashboard 加 Moments module 漏斗图。
- **#10 EAS Update** — `sentori.init` 自动 require `expo-updates`；event 加 `bundle: {id, deployedAt}`；server 索引 `bundle_id`；dashboard releases 加 bundle 子列。
- **#3 启动崩溃防护** — 原生写 `launch_marker.json`；JS 端 init 完成写 `launch_completed.json`；检测连续未完成 ≥ 2 → 触发 callback；blocking ≤ 200ms（D3 决策）。
