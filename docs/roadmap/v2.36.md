## NEVER rule

- Server-side `admin_get_push_send_detail` extended additively — `acked_at` / `ack_session_id` / `campaign_id` / `template_id` / `audience_tag` / `preference_category` all join the response payload.
- `admin_list_push_sends` query extended with `acked_at` + `campaign_id`; existing columns unchanged.
- Web types extend the existing `PushSendDetail.send` + `PushSendRow` shapes — same query keys, same fetch hooks; new fields are nullable so older server responses still parse cleanly.
- No new endpoints in v2.36 itself; this is a purely "make existing data visible" pass.
