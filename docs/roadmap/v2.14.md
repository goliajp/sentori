## NEVER rule

- **Zero SDK change.** Traces 数据 path 是 server `spans` 表 + `api/traces.rs` 已 alive;Live debug 是 SSE stream from `api/live_debug.rs` + Valkey live targets。SDK 不动。
- **Zero server change.** 两个 endpoint set 都自 v0.x alive。
- **Wire shape 不变.** `listTracesPage` / `getTraceDetail` / `/admin/api/projects/:id/live-debug/users/:id/{arm,DELETE,SSE}` 全保留。
- **Lens stop-ship rule.** Traces 归 `find-slow` group(audit verdict 已写 "find-slow fallback"),作 vitals/runtime 不够时的 drill-down。Live debug 归 `find-bug` group(admin utility,不是 lens module 但放进 find-bug group 因为最常的 use case 是"复现某个 user 的 bug")。两者归不同 lens 但都不需要新设计 lens — Live debug 是 utility,traces 是 fallback。**这跟 v2.13 Privacy 单独 ship 不冲突**:Privacy 是 *new lens anchor*,这两个是 *existing lens extensions / utilities*。

## BI-five answers per module

**Traces**(find-slow fallback)

1. **dim** — 一行 per root span (`trace_id`)
2. **measure** — duration_ms / span_count / root_op / root_name / last_seen
3. **drill** — row → `/traces/:traceId` detail 看 spans timeline
4. **compare** — n/a(每 trace 唯一,跨 trace 比较是 vitals 的 job)
5. **cache** — server `api/traces.rs` query;React Query 默认 staleTime

**Live debug**(find-bug utility)

1. **dim** — events of a specific armed user.id,实时流
2. **measure** — receivedAt / kind(error/anr/nearCrash)/ errorType / errorMessage / release
3. **drill** — n/a(stream 是 self-contained drill 视角)
4. **compare** — n/a
5. **cache** — SSE stream + 200-row client-side ring buffer
