## NEVER rule

- **Zero wire-shape break.** Same `/v1/push/*` routes, same `device_tokens` table. Series-wide guarantee held.
- **Same `sentori.push.register()` JS API.** Framework hooks are thin re-exports over `@goliapkg/sentori-core` types; no new client APIs.
- **CLI mirrors REST 1:1.** Every `sentori push *` command corresponds to an existing `/admin/api/projects/:id/push/*` REST call; no new admin endpoints needed.

## What v2.12 does NOT do

- No new server endpoints. CLI uses existing `/admin/api/projects/:id/push/credentials` etc.
- No dashboard changes. Operators upload HCM/MiPush creds via the same v2.11 module.
- No standalone HCM/MiPush recipe pages — short sections in iOS/Android recipes is enough.
