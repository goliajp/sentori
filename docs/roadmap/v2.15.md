## Lens verdict: Moments = find-slow

Audit doc 之前留 "find-user OR engineering-hygiene 待定"。仔细看 data:
- `MomentRow` = `{count, p50Ms, p95Ms, abandoned, failed, lastSeen}` per moment name
- `MomentSample` = `{startedAt, durationMs, status, abandoned}` per individual flow execution

Fields 是 latency + abandon rate + count — 这是 **business-flow performance**。SDK API 是 `sentori.startMoment('checkout').end()` — user 主动标注的 flow 边界,measuring "我的 checkout 用户走得快不快/有没有放弃"。

跟 v2.5 vitals(TTID/TTFD/frame drops,设备技术 vital)同**一个 lens 不同抽象层**:vitals 是 device 层,moments 是 business-flow 层。Find-slow operator 问"为啥我的 app 慢"自然往两层都要看。

放 `find-slow` group(vitals / runtime / metrics / moments)。Chord `m` (moments)。

## BI-five answers per module

**Moments**
