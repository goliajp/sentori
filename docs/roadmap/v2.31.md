# Sentori v2.31 — development plan

Status: **in progress 2026-06-10 — topic pub-sub fanout.** Twelfth version of Push Phase 2. Devices subscribe to N topics; `send` addressed to `to: { topic: '<name>' }` fans out to every subscribed device in the project. Skips v2.31-v2.33 in the original linear plan — those (RN local schedule / iOS critical+VoIP / Live Activity) are RN-native-heavy work and tracked as a future "Sentori RN Polish" series; v2.31 here is what was originally v2.34.

Owner: claude + takagi
Date: 2026-06-10

## NEVER rule

- **Zero wire break.** `to: { topic }` is one shape variant of the existing `to` union; non-topic sends behave exactly as v2.30.
- **Per-project isolation.** Same topic name across projects is independent — fanout filters on `device_tokens.project_id`.
- **Quiet on unknown topic.** A send to an unsubscribed topic enqueues zero rows and returns an empty `tickets` array — no error.
