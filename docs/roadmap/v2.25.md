## NEVER rule

- **Zero wire-shape break.** New fields are additive, optional, namespaced under `_sentori`. Legacy SDKs ignore unknown JSON keys → silent no-op. Existing public ingest API + admin endpoints + Expo-compat all preserve their pre-v2.25 shape.
- **Zero customer-side change required.** Server updates inject `_sentori.msgId` automatically; SDK side reads in v2.26. Customers running v1.x SDKs continue working unchanged.
- **Default behaviour preserved.** Customers that don't pass `campaignId / templateId / audienceTag` see exactly v2.24 behaviour — fields land as `NULL` in `push_sends`.
- **DB migration backward-compatible.** All new columns nullable. Old `push_sends` rows interpret as `NULL` — semantically "untagged".
- **Payload budget respected.** `_sentori.msgId` is ~30 bytes; APNs/FCM 4 KiB cap is unaffected (< 1 % budget).

## What v2.25 does NOT do

- **No SDK changes.** RN/JS/Next/etc. all stay on v2.24 wire shape (and continue to work). Reading `_sentori.msgId` + auto-breadcrumb + auto-track happens in v2.26.
- **No `events.push_origin_msg_id` FK.** That's the v2.27 schema. v2.25 only writes the primitive into outgoing payload; the JOIN target on the events table comes when the SDK round-trips it.
- **No dashboard surface for campaign/template/audience.** v2.27 BI slice work — until then these are write-only labels.
- **No bulk-tag-existing-rows migration.** Pre-v2.25 push_sends rows stay NULL forever; no fictional tag inference.

## Risk + mitigation

- **`_sentori` namespace collision.** Customer code that happens to use `data._sentori` is overwritten. Likelihood: ~zero (no documentation has ever referenced this), but documented in design doc rule #4 as a reserved namespace.
- **Payload size cap interaction.** Send gate (v2.20) checks `payload_size <= 4 KiB`. We inject _after_ that check, adding ~30 bytes. APNs's actual 4 KiB ceiling still applies after injection; if a customer maxes the budget today, the injected `_sentori.msgId` could push them over by 30 bytes. Mitigation: lower send-gate cap by 64 bytes in this version (`PAYLOAD_MAX_BYTES = 4 * 1024 - 64`) to leave headroom for injection.
