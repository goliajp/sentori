# Sentori v2.26 — development plan

Status: **in progress 2026-06-10 — RN SDK auto-correlation + confirmed delivery ack.** Seventh version of Push Phase 2. First version to touch SDK code in Phase 2. ## NEVER rule

- **Zero wire-shape break.** Same `/v1/push/*` routes plus one new additive route (`POST /v1/push/sends/:id/ack`). DB migration adds two nullable columns.
- **SDK upgrade is opt-in.** Existing v2.25-and-earlier SDK builds continue to work — they never read `_sentori.msgId`, never call ack, never write a `push` breadcrumb. Upgrading the SDK enables all three automatically with no host-app change.
- **No host-app perf hit.** Ack POST is batched 5s in the background; breadcrumb push is O(1) in-memory; auto-tracked event reuses the SDK's existing event pipeline. Receive path stays < 5 ms main-thread.
- **Native-side persistence reused.** v2.9 / v2.10 already buffer received notifications in a 32-slot FIFO at the native side (iOS UNUserNotificationCenter delegate + Android FirebaseMessagingService). v2.26 only adds JS-layer processing on the drained items — no new native code, no Info.plist / AndroidManifest changes.

## What v2.26 does NOT do

- **No JS web SDK (`sdk/javascript`) auto-pipeline.** Service Worker push event integration is deeper (different handler shape, postMessage to main thread). v2.26.1 / v2.27 picks it up.
- **No `events.push_origin_msg_id` FK.** That's v2.27 — needs the events ingest path to read the breadcrumb + thread it through.
- **No "Open" / "Tap" deep-link routing changes.** Existing tap callback path (v2.9) unchanged; v2.26 just observes.
- **No backfill of pre-v2.26 push_sends.** They stay un-acked; correlation queries treat NULL as "unknown delivery state".
- **No `iOS NSE` / `Android BigPicture` work.** That's v2.28.

## Risk + mitigation

- **Ack batching loses on app kill.** If the host kills before the 5 s tick, the ack is lost. Acceptable — server-side `push_sends.sent_at` is still set; the missing ack just means correlation degrades to "we know we tried to deliver" rather than "we know it arrived". For higher fidelity, host can opt into immediate flush.
- **BreadcrumbType union drift.** `CLAUDE.md` lockstep rule + CI will catch a missed half (server side validates incoming `breadcrumbs[].type`). We're updating both in this single commit.
- **Ack endpoint abuse.** Public Bearer endpoint — a malicious client could spam acks for arbitrary send_ids. Mitigation: idempotent on first-ack-only (subsequent acks no-op). Rate limit reuses the existing `/v1/push/tokens` chain.
