## NEVER rule

- **Zero wire-shape break.** Same `/v1/push/*` routes, same `device_tokens / push_credentials / push_sends / push_delivery_logs` schemas. No additive columns in v2.20 (correlation FKs come in v2.27).
- **Zero customer-side change required.** Customers update server image and inherit every improvement. No SDK update needed. No credential reconfiguration.
- **Out-of-the-box defaults.** TokenCache TTL / retry classification / send-API caps all ship with sensible production defaults. Configuration is for advanced override, not daily operation.
- **Host-app performance untouched.** v2.20 is server-only. No SDK code changes.

## What v2.20 does NOT do

- **No new wire fields.** `_sentori.msgId` primitive and `campaignId/templateId/audienceTag` come in v2.25; correlation FK comes in v2.27.
- **No SDK code changes.** Every SDK (`sdk/react-native`, `sdk/javascript`, `sdk/expo`, `sdk/next`, framework hooks, cli) is untouched.
- **No new admin endpoints / dashboard views.** Send inspector is v2.24.
- **No DB migration.** Schema unchanged.
- **No per-provider client isolation.** That is v2.21 (uses one shared `reqwest::Client` for now — fine until quarantine logic needs per-provider pools).
- **No three-layer rate limit.** v2.20's `send_gate` is only the input-side guard; the dispatch-side L1/L2/L3 limiters are v2.22.
- **No invalid-rate health gauge.** v2.23.

## Risk + mitigation

- **TokenCache test injection.** Production code path takes `&self` and uses a real `Clock`; tests inject a `MockClock` via constructor. Avoids panicking on `Instant::now()` in test contexts.
- **Smart retry behaviour change.** Existing retry schedule constants are preserved as the `Transient(None)` ladder. Customers see identical retry cadence except when provider supplies `Retry-After`, where Sentori now honors it (improvement, not regression).
- **Send gate default of 60/min/token.** Conservative; in practice no legitimate use case sends more than a handful per minute to one device. If a customer reports `TokenRateLimited`, it is almost certainly a bug in their integration — exactly the kind of latent client bug Sentori as observability tool should surface.
- **`VERSION` file rename.** Root-level `VERSION` is a common convention (Rust, Linux, dpkg). No existing `VERSION` file in this repo (confirmed).
